Lesson 1: The Basics of the Web and HTML


How The Web Works
The web is a network of computers that communicate with each other. When you visit a web page, like www.google.com, your computer sends an HTTP request to a server. The server locates the appropriate HTML document and gives it to your web browser. The browser interprets the page and displays the page to your screen.
  • Invented in the 1990s
  • The world's largest computer network
  • Now has 30 billion pages
HTML
HTML stands for Hyper Text Markup Language
  • A web page is a text document written in HTML
  • HTML is the main document on the web and makes up the heart of the web
  • HTML consists of:
    • Text content- what you see
    • Markup- what it looks like
    • References to other documents, images, and video
    • Links to other pages
Tags and Elements
HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags.
  • HTML Markup: <name> contents </name>
    • <name> is called the opening tag
    • </name> is called the closing tag
    • The element refers to the opening tag, closing tag, and what is in between the tags.
  • Example elements:
    • <b> Creates bold text.</b>
    • <em> Creates emphasis or italicized text. </em>
    • <br> Creates a break (blank line).
      • This is a void tag because it does not have a closing tag.
  • HTML grouping tags:
    • <span> Group in-line elements, no visual change
    • <div> Group block elements to be formatted with CSS
      • It is good practice to use <div> with each "imaginary box"
  • HTML attributes are specified in the opening tag and provide additional information about the element
    • <tag attr="value">contents</tag>
    • <a href="www.udacity.com">Udacity</a>
    • <img src="url" alt="text">
      • alt: Specifies alternate text. This is useful for broken requests and visually impaired users.
Why Computers are Stupid
Computers are often referred to as "stupid," because they can only do what they are programmed to do. They are often very unforgiving of small mistakes made by a programmer. Programmers use code to write in a language the computer can understand. This allows the computer to execute instructions written by the programmer. No programmer writes perfect code the first time, this is where "debugging" comes into play. Debugging is the process of finding and correcting problems in the code.
Inline vs Block Elements
Inline element examples:
  • <a> Defines a hyperlink
  • <br>Inserts a single line break
  • <img>Defines an image
  • <strong> Defines important text

Block elements form an "invisible box" around the content. Example:
  • <p> paragraph </p>

Lesson 2: Creating a Structured Document with HTML


Front End Programming
Front end programming is the visual aspect of a website. Some front end languages are HTML, CSS, and Javascript.
A website is much like a house:
  • HTML controls the structure, like where walls are placed.
  • CSS controls the style, colors, etc.
  • Javascript controls the interactive components, like a garage door.
The "tree-like structure" of HTML
Document Object Model (DOM): The "tree-like structure" of HTML

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...
The relationship between indented HTML and boxes
When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.
Text Editors (for programming)
When writing code, programmers use special text editors (like Sublime Text for example). These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.
Layout of an HTML Document 
									
		<!DOCTYPE HTML>      (this declares the document type)
		<html>
			<head>
				<title> Title! </title>        (this specifies the title of the document)
			</head>
			<body> 
				<b> content </b>
			</body>
		</html>
Between the <head> tags, you specify the meta-data, Javascript, and CSS involved.
Between the <body> tags, you specify the body/content of the page.

Lesson 3: Adding CSS Style to HTML Structure


Avoiding Repetition
Avoiding repetition is important for a variety of reasons.

Avoiding Errors: When programmers don't avoid repetition, they will often have to do the same thing over and over. This may mean copy and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times. This can lead to errors when the programmer decides to make a change to something. If they don't dilligently make that same change everywhere the repeated code appears, problems will arise.

Consistency: Easily create pages of the same style without manually doing each page. Assign similar "boxes" with the same class and then specify the style for that class in only one place! By avoiding repetion, it's very easy to make style changes.
CSS
CSS stands for Cascading Style Sheets. This uses the idea of inheritance and gives programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.

There are three methods of adding CSS to your HTML:
  • Write CSS in th <head> of your HTML. Easiest method for a small project.
  • Link your HTML to a separate CSS file. Requires an additional step but keeps you organized.
  • Write your style inline with your HTML. This tends to lead to repeated code and more errors.

Computers & Programming
A computer is a machine. Unlike other machines that have a specific purpose (like a toaster), computers have many abilities. Programming is the act of giving a computer precise directions so that it will perform what we need it to do. An advantage of programming a computer is the speed at which it can run computations. By writing programs, we can make the most of a computer's ability.

Python is a high level programming language. One of the advantages of a high level language is the relative ease of use. Python uses a program called an interpreter. The interpreter converts the code into a language that the computer can understand. The interpreter also allows for portability of code across multiple systems.

The purpose of Backus-Naur Form& is to describe a language simply and concisely. This is the example of an idea used in computer science.

We can break down the langage into two primary types:
  1. Non-terminals (something that isn't finished)
  2. Terminals (something that is completed)


Non-terminals can be paired with terminals or other non-terminals. Non-terminal rules dictate which other non-terminals it can be "linked" with.

Lesson 5: Variables & Strings


The Variable
Variables make our code easier to read and more expressive. A variable is the name for a location in a computer's memory. Using a variable name such as "miles_per_hour" in place of the number "60" increases the readibility of your code. A variable can be given a value of many data types.

In the example: "miles_per_hour = 60"
  • Variable is "miles_per_hour"
  • Value is "60
  • Assignment operator "=" gives the variable a value.
    • Should not be confused with the equals sign in arithmetic.
The String
A string is a sequence of characters. In Python, a string can be written between single or double quotes.

One advantage of strings over numbers is the feature of indexing. Indexing allows us to use square brackets "[ ]" to select a character in a string. If there is a variable that contains the string "udacity" and your code includes the line "print udacity[3]" it will print the fourth character "c".

You can also select a substring, which is a subsequence of characters within the string. This is done by placing a colon between two index positions, "udacity"[0:4]. If the string "udacity" has the value 123456, the code would print "1234."
The Find
The Find method allows us to locate substrings within a string. To use Find, you provide the method two strings. It will return the first index of the "target string". If the string is not found, it returns -1. You can pass a number in the second parameter of Find, this causes Find to begin searching at that index.

Lesson 6: Input-Function-Output


The Function
A function is a set of instructions defined by the programmer and is an important part of a program. It allows for readability and reduces redundancy in your code. When defining a function, we list arguments that can be passed to the function. The arguments are like variables used only inside of the function. To use a function, you only need the name and the number and type of arguments you can safely pass.

Lesson 7: Decisions & Repetition


Comparisons
Comparison operators in Python are used to compare numbers. The comparison will return a Boolean value of True or False. Some examples of comparison operators are: equal to, is not, less than, and more than.
The If
The if statement tests a given expression to decide if it is True or False. If True, the code inside the if statement will run. If False, the program will skip over the rest of the if statement and advance to the next code.
The Or
The Or Operator in Python allows us to test expressions to see if one evaluates to True. The expression that satisfies the condition is not returned, but it allows the rest of the code to run. If none of the conditions are true, they are skipped.
The Loop
Loops allow us to write a block of code once, but run it multiple times depending on a condition. An example is the While Loop. The While Loops executes the block of code as long as the test expression resolved to True. Once the expression becomes False, the loop is ended and advances to the next code.

Lesson 8: Structured Data


Lists
Unlike a string, which is a sequence of characters, a list can be a sequence of any data type (Even a list of lists!). Just like a string, we have functions that allow us to select a subsequence of the list. Another function we can perform on a list is Mutation. This allows us to modify the contents of the list even after it is written to memory.
List Operations
There are many operations we can perform on a list. For example, we can "append" a list, which simply adds an element to the end of the existing list. The "+" can be used to tack a list on to the end of another list without the need for nesting. The "len" feature can be used to return the number of elements within the specified list. Additionally, we can also use a For Loop to iterate through the list and execute a block of code on the individual element.

Lesson 10: Classes and Objects


Classes and Inheritance

The idea of a class can be seen as a categorization of things that share similar attributes.


"The class is like a blueprint of a building, where-as the object is the actual building that's been built according to the blueprint."


I visualize the class-object relationship like a parent-child relationship. The child object will always inherit its parents' attributes.


My Analogy
I like thinking about vehicles to understand objects and classes. A vehicle has many standard functions like turning on the lights, starting the engine, driving, and stopping. There can also be different properties, such as color, number of wheels, number of cylinders, etc. There can be many instances of a vehicle, for example a compact car, a sedan, a truck, a semi, etc.
Class Constructor

Python has pre-defined classes, but we can create our own classes. A class definition starts with a function named __ init __ (a constructor), and takes in some attributes of the object as its arguments.


"A class is a template for objects. A class defines object properties including a valid range of values, and a default value. A class also describes object behavior. An object is a member or an "instance" of a class. An object has a state in which all of its properties have values that you either explicitly define or that are defined by default settings."


Instance variable: A variable that is defined inside a method and belongs only to the current instance of a class.


Class variable: A variable that is shared by all instances of a class. Class variables are defined within a class but outside any of the class's methods. Class variables aren't used as frequently as instance variables are.


Defining Documentation: You can define documentation for your class by using triple quotes, such as: """This is my documentation""" This would be held in the pre-defined class variable __doc__.


Reusing Methods: Inside a class, you may define a method. This method then can be accessed by the parent and child instances.


Method Overriding: If you have created a method under a parent class, such as "show_info", you can define a method of the same name within the child's class. When the child's method is called, it will not run the show_info method that it inherited from the parent, instead it will run the overridden method.

Lesson 11: Networks and Databases


Basic Vocabulary
  • Network: a group of entities that can communicate without being directly connected
  • Latency: time it takes a message to get from source to destination
  • Bandwidth: amount of information that can be transmitted per unit time
  • Protocol: a set of rules that allows two entities to talk to each other (i.e. HTTP)
  • Internet Protocol (IP) address: a numerical label assigned to each device participating in a network. The IP address has two main functions, location addressing and identify host or network interface. Example: 172.16.254.1
  • Uniform Resource Locator (URL): the human readable locator (like www.example.com) which resolves to the numerical IP address.
How Networks Work:
    The web browser determines which protocol should be used. Examples of protocols include File Transfer Protocol (FTP) and Hypertext Transfer Protocol (HTTP). To retrieve a document via HTTP, the browser sends the following request to the server: GET/request-URI HTTP/version. Example GET request with headers below: 
    
						GET /course/viewer HTTP/1.1
						Host: www.udacity.com
						User-Agent: Chrome v17
    The server now responds to the browser's requests. It verifies that the given address exists, finds the necessary files, runs the appropriate scripts, exchanges cookies if necessary, and returns the results back to the browser. The server returns a response consisting of a line with a status code, a list of header fields, a blank line, and then the requested document. If the server cannot locate the file, it sends an error message to the client. Example of a server’s response: 
    	
						HTTP/1.0 code text
						Field1: Value1
						Field2: Value2
						
					...Document content here...

    The result the server sends is either static or dynamic. Examples of a static response are prewritten files and images. An example of a dynamic response is a web application. Once the browser receives the data, it translates the data into HTML and displays the results to the user. The browser translates the data it has been given in to HTML and displays the results to the user.


    GET Headers:
  • User-Agent: This is a string identifying the user agent.
  • Referer: The referer field tells the server where the user came from, which is very useful for logging and keeping track of who links to the webpage.
  • If-Modified-Since: It the browser already has a version of the document in cache, it can include this field and set it to the time it retrieved the document.
  • Authorization: This field can hold username and password if the document requires authorization.

    • Headers sent by server:
  • Location: This tells the user agent the URL where the resource it requested can be found.
  • Server: This tells the user agent which web server is used.
  • Content-length: The size of the resource, in bytes.
  • Content-type: This describes the file format of the resource.
  • Last-modified: This tells the browser when the resource was last modified. Can be useful for mirroring, update notification etc.
Validation

Data validation is the process of checking user input to see whether it is in the expected format and is an allowed value. It has a number of purposes, some of which are below. If the user inputs data that is not within allowed values or is in the wrong format, it could cause the application to execute unexpected behavior. Validation allows for this to be prevented and instead to present a human-readable error message back to the user. Allowing the user to see why the input wasn't accepted greatly aids in usability of the application. Validation can help to ensure that the data stored is complete and nothing is missing. For instance, ensuring that 'required' fields are filled out by the user makes sure that there won't be gaps or empty strings in a record, which could cause problems with the incomplete data is acted upon later.

Examples of Validation
  • Make sure the value is a whole number(integer).
  • Make sure the value is numeric and isn't an arbitrary string.
  • Make sure the value sits between the highest value and the lowest value.
  • If there are some values which aren't allowed(such as zero), check that it is not one of those values.
  • Make sure string is a valid length.
  • If string is "required", make sure it isn't blank.
  • Make sure the value is equal to one of the allowed choices.
  • Make sure the value is true or false. Example: 0 or 1

Functions in Javascript

We learned in Python that functions are like mini-programs that allow code reuse and create more readable code. Javascript also allows functions. A function literal consists of four pieces of information. The first item is the word ‘function,’ which is a reserved word to declare that this is going to be a function. The next item is the name of the function (interestingly, this is optional and functions can be anonymous…) The third part of a function is the parameter list. The last bit of the function is the content, or the statements to be invoked. These statements are surrounded by { }. 

					function name (parameterA, parameterB) {
						statements;
					}

When invoking a function, the number of arguments provided does not have to match the number expected. This is different than other programming languages. If the arguments provided are less than what is expected, the function assumed the value is undefined. If the number of arguments is greater than what is expected, the additional arguments are ignored.

Functions must return a value. If not specified by the programmer, the function will return undefined.

Exceptions

Try-catch blocks are also used to catch exceptions. The Try block attempts to fun the function, and if an exception occurs, control is passed to the catch block which handles the exception.

The throw statement is used to handle possible exceptions like incorrect datatypes. When this occurs, the throw stops the execution of the code. This statement consists of a “name” which is the type of exception, then a “message” that explains the error that occurred. A throw statement may look like this:

					throw {
						name: 'TypeError';
						message: 'Function requires numbers';
					}

It appears like [a, b, c, d], where a’s propery name is 0, b’s is 1, etc. The array has a “length” property which allows us to find the number of elements. Length increases as elements are added to the array. Arrays in Javascript also have a useful push() method which allows us to add an element to the end of the array. And conversely, pop() allows us to remove the last item. Delete() allows us to remove an object from the array, however an empty spot is left. An alternative is to splice() the array. Javascript only has one dimensional arrays, but can have arrays of arrays.


There are two ways to create an array in Javascript.

Objects

Objects may have properties, methods, and events. A property is simply a variable that is associated with the object. The method is a function that the object is able to perform at the request of the user. Events trigger responses by the object and are specifically useful in web design. For example, you can allow an image to change when the mouse hovers over it. You will notice this is a feature I have implemented in the navigation menu of my website. I chose to do this because it suited the needs of my website and also developing a menu seemed like a good way to learn the basics of Javascript.

How to add Javascript to your website:
Javascript code is placed between the script tags. Javascript can be called from an external file.

Why use Javascript and how does it relate to web design?

  • 1. HTML defines a web page's content
  • 2. CSS specifies the layout of the page
  • 3. JavaScript controls the behavior of the web page

One useful method in javascript is .getElementById(), which I made use of in my website. This method finds an HTML element with a particular ID and then performs some action with it. Javascript can display data by opening a dialogue window (window.alert()), writing to the HTML (document.write() or innterHTML), or writing into the browser console (console.log()).

Javascript Events

An event is something the browser or user does. In HTML, an event may be the change of an input field or the clicking of a button. We are able to execute code at the occurrence of one of these events.

Examples of HTML events:
  • onchange: when an element changes
  • onclick: when the user clicks the element
  • onmouseover: when user moves mouse of the element
  • onmouseout: when user moves away from the element
  • onkeydown: when user pushes a key
  • onload: when page finishes loading
